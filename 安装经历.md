# 📝 安装经历

记录小贾智能助手的安装和配置过程

---

## 遇到的问题

| 问题 | 解决方法 |
|------|----------|
| Windows 10 老版本无法运行安装命令 | 升级到 Windows 11 或更高版本 |
| Windows 11 上直接安装 Node.js 失败 | 使用 WSL（Ubuntu 22.04） |
| WSL 网络通信问题 | 放弃 WSL 方案 |

---

## 安装过程

### 1. Windows 10 尝试

最初想在 **Windows 10**（老版本）上安装小贾，运行安装命令时发现无法运行。经过排查，发现是 **Windows 版本太低**导致的兼容性问题。

### 2. Windows 11 尝试

后来在 **Windows 11** 机器上尝试安装，本来想直接安装，但发觉直接在 Windows 系统上安装 Node.js 是不行的。

### 3. WSL 尝试

于是尝试使用 **WSL**（Windows Subsystem for Linux），也就是 Linux 系统。WSL 是 Ubuntu 22.04 版本。

在 WSL 中可以通过命令行直接安装小贾，安装成功了。

但是存在一个问题：**WSL 和 Windows 宿主机之间的端口映射和网络通信存在问题**。

### 4. 最终方案

由于 WSL 的网络问题，最终 **放弃了 WSL 方案**。
