# 📝 安装经历

记录小贾智能助手的安装和配置过程

---

## 遇到的问题

| 问题 | 解决方法 |
|------|----------|
| Windows 10 老版本无法运行安装命令 | 升级到 Windows 11 或更高版本 |
| Windows 11 上直接安装 Node.js 失败 | 使用 WSL（Ubuntu 22.04） |

---

## 解决方法

### Windows 系统兼容性问题

最初想在 **Windows 10**（老版本）上安装小贾，运行安装命令时发现无法运行。经过排查，发现是 **Windows 版本太低**导致的兼容性问题。

**解决方案：**
- 没有继续升级 Windows 10，而是选择在 **Windows 11** 机器上运行

### WSL 安装方式

在 **Windows 11** 机器上，直接在 Windows 系统上安装 Node.js 失败。

**尝试的方法：**
- 直接在 Windows 上安装 Node.js → 无效
- 使用 WSL（Windows Subsystem for Linux）→ 成功！

**WSL 环境：**
- 系统：Ubuntu 22.04
- 成功安装并运行小贾

### WSL 的网络问题

WSL 方式可以成功安装小贾，通过命令行即可安装。

**但存在问题：**
- WSL 是运行在 Windows 上的 Linux 子系统
- WSL 和 Windows 宿主机之间的**端口映射和网络通信存在问题**
- 无法正常访问 WSL 中运行的服务

**因此放弃了 WSL 方案**
